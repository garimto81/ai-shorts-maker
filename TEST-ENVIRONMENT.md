### 1. 개발 서버 실행
```bash
npm run dev
```

## 🎮 테스트 시나리오

### 시나리오 1: TTS 기본 테스트
1. 테스트 환경 접속
2. "시스템 체크" 실행 → API 상태 확인
3. "샘플 이미지" 생성 → 5개 테스트 이미지 생성
4. "TTS 테스트" 실행 → 음성 생성 및 재생 확인
5. 메인 페이지에서 전체 워크플로우 실행

### 시나리오 2: 파일 업로드 테스트
1. 음성 파일 준비 (MP3, WAV 등)
2. 이미지 파일 준비 (JPG, PNG 등)
3. 테스트 환경에서 "업로드 테스트" 실행
4. 메인 페이지에서 "음성 파일 업로드" 모드로 전체 워크플로우 실행

### 시나리오 3: 에러 처리 테스트
1. 잘못된 파일 형식 업로드 시도
2. 너무 큰 파일 업로드 시도
3. API 키 없이 TTS 생성 시도
4. 에러 메시지 및 복구 가이드 확인

## 🐛 문제 해결

### 자주 발생하는 문제

**1. TTS 음성이 생성되지 않을 때**
- Azure Speech API 키 확인
- 관리자 설정에서 API 상태 확인
- 네트워크 연결 확인

**2. 파일 업로드가 실패할 때**
- 파일 형식 확인 (지원: MP3, WAV, M4A, AAC, OGG)
- 파일 크기 확인 (최대 50MB)
- 브라우저 콘솔 에러 확인

**3. 비디오 렌더링이 실패할 때**
- FFmpeg 설치 확인 (개발 환경에서는 자동)
- 시스템 리소스 확인
- 임시 파일 저장 공간 확인

### 디버깅 팁
- 브라우저 개발자 도구 콘솔 확인
- 테스트 환경의 실시간 로그 활용
- 관리자 설정의 시스템 상태 모니터링
- 네트워크 탭에서 API 요청/응답 확인

## 📊 성능 벤치마크

### 예상 처리 시간
- **이미지 업로드**: <5초 (5개 이미지)
- **TTS 생성**: 10-30초 (텍스트 길이에 따라)
- **스크립트 생성**: 5-15초
- **비디오 렌더링**: 30초-5분 (품질/길이에 따라)

### 시스템 요구사항
- **메모리**: 최소 4GB, 권장 8GB
- **저장소**: 최소 1GB 여유 공간
- **네트워크**: API 호출을 위한 안정적인 인터넷 연결
